1. Поясніть етапи завантаження системи Linux.
BIOS / UEFI → GRUB → Kernel → initramfs → systemd → user space

BIOS / UEFI — перевіряє залізо, знаходить що запускати
GRUB — вибирає ОС, завантажує kernel + initramfs
Kernel — ініціалізує систему, драйвери, монтує root FS
initramfs — тимчасова FS, допомагає знайти root
systemd — запускає сервіси
user space — логін / GUI

2. Як переглянути системні журнали?

Основне:
journalctl
journalctl -b        # поточний запуск
journalctl -u nginx  # конкретний сервіс
journalctl -f        # в реальному часі
journalctl -p err    # тільки помилки
Додатково:
dmesg               # kernel
/var/log/           # файли логів

3. -rw------- : Опишіть цей дозвіл. Як додати прапорець виконуваного файлу?

-rw------- — що це означає
- файл, не папка
-rw- доступ на читання і запис у власника файлу
--- група немає доступу
--- інші не мають доступу

chmod u+x filename.txt  # додається право запускати файл для власника файлу

4. Яка різниця між apt і dpkg?

dpkg

старий низькорівневий інструмент
працює з .deb
не вирішує залежності

apt більш новий, високорівневий інструмент
встановлює пакети з репозиторіїв
вирішує залежності
